# 支付宝接入步骤 (商户已审批通过后)

代码侧已经完成 (alipay.py 已实现, main.py 加了 notify 端点, 前端拦截解除).
你只需要在**支付宝开放平台**走 5 步, 拿到 4 个 env 填到 .env 即可.

## Step 1: 进开放平台创建/检查应用

进 [open.alipay.com](https://open.alipay.com) → 登录商户账号 → **控制台** → **网页&移动应用**

如果还没创建应用:
- 点 **创建应用** → 类型选 **网页/移动应用** → 应用名 "monoi" → 应用图标 (用 monoi logo)
- 提交后等 1-2 天审核 (跟商户开通是分开的, 别混)

如果已经创建过:
- 直接进入 → 复制 **AppID** (16 位数字, 类似 `2021004105696589`)

→ **这是 `ALIPAY_APP_ID`**

## Step 2: 给应用绑定 "当面付" 能力

应用详情页 → **能力管理** (或 "添加能力") → 找到:
- **当面付** ✅ 必须勾选 (我们用 `alipay.trade.precreate` 扫码下单)
- ~~网页支付~~ 不用 (UI 选了扫码方案)

签约条件:
- 营业执照 (你已经有)
- 提供网站 / 业务介绍 — 用 monoi.cn

支付宝审核 1-3 天.

## Step 3: 生成 RSA2 密钥对

下载 **支付宝开放平台密钥工具**:
- Windows: https://opendocs.alipay.com/common/02kipl
- macOS: 同上

打开工具 → **生成密钥** → 选 **RSA2 (推荐)** → **PKCS1 (非 Java 适用)**

工具会生成两个文件 (一般在 `桌面/RSA密钥/`):
- `应用私钥2048.txt`  ← 留到本地, 这是你的私钥, 绝不能发给任何人
- `应用公钥2048.txt`  ← 这个要传到支付宝, 不算秘密

**打开 `应用私钥2048.txt`** — 整段内容 (包括 `-----BEGIN ...` 和 `-----END ...`) → **这是 `ALIPAY_APP_PRIVATE_KEY`**

## Step 4: 在开放平台上传你的公钥, 拿支付宝公钥

进应用详情 → **开发设置** → **接口加签方式** → 点 **设置/修改** → 选 **公钥**

- 把刚才 `应用公钥2048.txt` 的内容粘贴进去 (不含 BEGIN/END 头尾, 只有中间那段 base64)
- 保存后, 同一页会显示一个 **"支付宝公钥"** (一整段 PEM)
- 点 **复制** 那一段

→ **这是 `ALIPAY_PUBLIC_KEY`**

⚠️ 注意区分:
- "应用公钥" 是**你的**, 上传给支付宝
- "支付宝公钥" 是**支付宝的**, 我们用来验签 notify

## Step 5: 设置支付/异步通知 URL

同一页 **开发设置** → 找到:
- **授权回调地址**: `https://monoi.cn/account` (用户支付后可能跳回的页面, 当面付不强制用)
- **应用网关 URL**: 可不填 (我们不接收异步推送, notify 走商户业务的固定 URL)

主回调 URL 不在这里配, 是在**每次下单时**通过 `notify_url` 参数传给支付宝.
代码里默认是 `https://monoi.cn/api/pay/alipay/notify` (alipay.py 第 26 行 `_DEFAULT_NOTIFY_URL`).

可选 env: `ALIPAY_NOTIFY_URL` — 想自定义的话设这个 (例如测试期改成 ngrok 内网穿透).

## Step 6: 安装 Python SDK + 填 env

Windows 服务器:

```cmd
cd C:\path\to\windows-server
pip install python-alipay-sdk-no-crypto
```

**注意**: 不要装 `alipay-sdk-python` (官方那个名字, Python 支持差). 装 `python-alipay-sdk-no-crypto`, 这是社区维护的, Windows 不需要 C 扩展.

编辑 `.env`, 加 3 行 (NOTIFY_URL 可省, 用默认即可):

```env
ALIPAY_APP_ID=2021004105696589
ALIPAY_APP_PRIVATE_KEY=-----BEGIN PRIVATE KEY-----
MIIEvAIBADANBgkqhkiG9w0BAQEFAASCBKYwggSiAgEAAoIBAQ...(几十行)...
-----END PRIVATE KEY-----
ALIPAY_PUBLIC_KEY=-----BEGIN PUBLIC KEY-----
MIIBIjANBgkqhkiG9w0BAQEFAAOCAQ8AMII...(几行)...
-----END PUBLIC KEY-----
# 可选: 不设走默认 https://monoi.cn/api/pay/alipay/notify
# ALIPAY_NOTIFY_URL=https://monoi.cn/api/pay/alipay/notify
```

⚠️ **PEM 多行格式** — 在 `.env` 里, 多行字符串需要写**真换行**, 不要用 `\n` 转义. 用记事本 / VS Code 打开 .env 粘贴时, 直接保留换行就行. uvicorn 启动会用 python-dotenv 读, 支持多行.

如果你的 .env 里多行有问题, 也可以把私钥/公钥写成单行 (去掉所有换行 + 去掉 BEGIN/END 头尾, 只剩中间一长串 base64) — 代码里的 `_normalize_pem` 会自动包头尾.

## Step 7: 重启后端测一笔

```cmd
cd C:\path\to\windows-server
# 杀掉旧 uvicorn (Ctrl+C 或 taskkill /F /IM python.exe)
uvicorn main:app --host 0.0.0.0 --port 8000
```

启动日志看到:
```
[alipay] mode = REAL
```
就成了. 如果是 `mode = OFF`, 看下面 env 缺哪个的提示.

测试:
1. 浏览器进 monoi.cn → 充值 → 选**支付宝**
2. 二维码渲染出来 → 手机支付宝扫码 → 付 0.01 元
3. 后端日志看到 `[pay] order ord_xxx 标记 paid`
4. 网页自动跳转 → 显示已开通

如果 notify 没收到, 不用慌 — 前端每 2 秒主动 `query_order`, 支付宝侧 paid 后我们也会同步标记 paid. 这是 wxpay 也有的兜底逻辑.

## 常见问题

### Q: 二维码显示出来但扫不出来

检查 alipay.py 日志, `qr_code` 应该是 `https://qr.alipay.com/bavh4wjlxf12tper3a` 这种格式. 如果是空字符串, 说明 precreate 接口报错, 看 main.py 日志的具体错误码:

- `40004` + sub `INVALID_PARAMETER` — 参数问题 (一般是 amount 格式)
- `40006` + sub `ACQ.MERCHANT_AGREEMENT_INVALID` — 没签当面付能力
- `20000` — 支付宝系统问题, 重试

### Q: notify 一直不通

支付宝 notify 来自支付宝服务器, 必须从公网能访问 `https://monoi.cn/api/pay/alipay/notify`. 检查:
- monoi.cn 域名解析正常
- nginx / Vercel 没拦 POST
- 后端确实跑着 (uvicorn 进程在)

测试期 monoi.cn 没备案打不开的话, 用 ngrok 临时穿透 → 把 `ALIPAY_NOTIFY_URL` 设成 ngrok URL.

### Q: 验签老失败

99% 是支付宝公钥贴错了 — 你贴的是**自己的应用公钥**, 应该贴**支付宝公钥** (Step 4 第二段). 工具生成的"应用公钥2048.txt"是上传给支付宝的, 不是用来验签的.

## 总结 4 个 env

```env
ALIPAY_APP_ID=2021004XXXXXXXXX
ALIPAY_APP_PRIVATE_KEY=<你生成的应用私钥, 含 BEGIN/END>
ALIPAY_PUBLIC_KEY=<开放平台显示的支付宝公钥, 含 BEGIN/END>
# 可选:
# ALIPAY_NOTIFY_URL=https://monoi.cn/api/pay/alipay/notify
```

填好重启 → 测 0.01 → 跑通了我们就可以一并发了.
