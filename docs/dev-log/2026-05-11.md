### Sentinel + Choice 系列 bug

**bug 1**: 合成完成后 AI 给的 "生成封面/生成发布文案/保留视频" 三选一, 点"生成封面"后 `__form_cover__` 被当文本发给 LLM, 不弹封面工具. 根因: `useChat.chooseOption` 看到 `__` 前缀就 send(), 没区分 "UI 触发" 和 "带 payload 给 LLM" 两类 sentinel.
- 修法: `useChat.chooseOption` 加 `FORM_SENTINELS` 白名单 (10 个表单触发 id), 命中后 `dispatchEvent('monoi:open-form')` + return, 不发 LLM. `ChatInput` 加 `useEffect` 监听, 复用现有 `pickModuleOption` 弹表单.
- `__dialect__` / `__digital_human_video__` 这类带 payload 的 sentinel 不受影响

**bug 2**: 修完 bug 1 后用户再点老对话里的"生成封面"还是不弹. 根因: `ChoiceButtons` 把已 `chosen` 的按钮 `onClick` 直接 `!chosen && onChoose(opt)` 拦掉了, 之前那次错发给 LLM 时 `chosen` 已写进 store, 永远进不到事件分发.
- 修法: 已选中的按钮 (`isChosen=true`) 允许再点 — 语义"重做这个选择" (重开表单/重发文本). 同组未选的按钮保持 disabled 避免触发别的 LLM 分支

**bug 3**: 老对话里 `__form_cover__` 已经作为 user msg 持久化在 zustand store 里, 修不掉历史. MessageBubble 加兜底过滤: user 气泡里 `/^__[a-z][a-z0-9_]*__$/` 形的纯 sentinel 文本块跳过不渲染, 整条消息全是 sentinel 时直接 return null

## 启动自动发布功能 (新模块)

### 调研
- 用户先发 [social-auto-upload](https://github.com/dreammis/social-auto-upload), 说"自动发布用这个"
- 调研: 支持抖音/小红书/B站/快手完整, **视频号代码未完成** (核心交互函数空的). Python + Patchright (Playwright stealth fork) 模拟浏览器, 手动扫码登录, cookie 文件持久化. 自带 `sau_backend.py` 是 Flask 服务监听 5409 端口可以作为发布执行器
- 用户后又说"小红书用呼出浏览器方式" 并指 `docs/agent-bootstrap.md` — 但调研发现那个文档其实是给 AI 编程助手的 prompt 模板, 跟浏览器调用方式无关, 仓库**没有 CDP attach 路径**
- 跟用户确认实际需求 = "弹一个浏览器让用户自己手动登录, profile 持久化, 之后自动上传" = Persistent user-data-dir 方案 (C)
- 又因为"Windows 国内不翻墙没 Chrome", 把方案从 `channel="chrome"` 改成 `channel="msedge"` (Edge Windows 自带, Playwright 一等支持)
- 仓库**没有 LICENSE 文件** (README 说"暂时 MIT", 但 GitHub 没识别), 直接 vendor 有版权风险. 决定: **不 vendor, 自己写 ~200 行直接用 Playwright + Edge**, 只覆盖我们需要的"开浏览器 → 登录 → 上传 → 填表 → 发布"这一条线

### Stage 1: 骨架 + 登录态验证 (已落地)
- `windows-server/social_publisher.py`: `launch_edge_persistent(profile_dir)` + `check_login(platform)` + `open_login_window(platform)` + `debug_page(platform)`
- `windows-server/test_publisher.py`: 用户在 Windows 上手动跑的 CLI (`login` / `check` / `debug` 三个命令)
- `windows-server/INSTALL_PUBLISHER.md`: 装环境 + 首次扫码 step-by-step
- profile 目录默认 `D:\monoi-server\edge-profile`, 单 profile 多平台共用 (XHS / 抖音 不同 domain, cookie 不冲突), 可 `MONOI_EDGE_PROFILE` 环境变量覆盖

### Windows 实测过程
- 用户拉新代码受阻 — `D:\monoi-server` 不是 GitHub clone, 没 origin 远端 (历史是手动拷贝的). 这次用 curl 单独拉 3 个新文件凑合, 长期事项是改成 clone 模式
- 装 `pip install playwright` + `python -m playwright install msedge` 顺利
- 小红书登录一次过, `check xhs` 返回 `logged_in: True`. 注: 输出里 `url_probed` / `url_final` 字段显示成了中文页面 title 而不是 URL (诡异, 可能是 Edge/Playwright 某种 quirk), 但判断逻辑不依赖那个显示, 结果正确
- **抖音第一次跑 `check douyin` 返回 `logged_in: False`** — 原因: DOM 文字探针 `text=发布视频` 没命中 (抖音改文案了). URL 实际停留在 `creator.douyin.com/creator-micro/content/upload` 没被 redirect, 说明 cookie 是有效的, 只是探针选错了
- 改 `check_login` 策略: 不再依赖 DOM 文字, 改用 **URL-based 判断**:
  1. final URL 含 `login` / `passport` → 未登录
  2. 出 `creator.{platform}.com` 域名 → 未登录 (抖音特别会跳 `www.douyin.com` 主站)
  3. DOM 登录蒙层兜底 (`LOGOUT_PROBES`)
- 加 `debug_page()` + `python test_publisher.py debug xxx` 命令, 将来 selector 失效时用 (截图 + dump 常见标识词)
